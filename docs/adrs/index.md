# Architecture Decision Records

ADRs document significant technical decisions — what we chose, why, and what we considered. Each ADR is immutable once accepted; superseded decisions link forward to their replacement.

## Log

| # | Title | Status | Date |
|---|-------|--------|------|
| [0001](0001-example-decision.md) | Example Decision | Accepted | YYYY-MM-DD |

## Status Key

- **Proposed** — under discussion
- **Accepted** — approved and in effect
- **Deprecated** — no longer recommended
- **Superseded** — replaced by a newer ADR (link provided in the record)

## Creating a New ADR

1. Copy `0001-example-decision.md` and increment the number.
2. Fill in the template sections.
3. Set status to `Proposed` and open a PR for team review.
4. Update this index table when merged.
